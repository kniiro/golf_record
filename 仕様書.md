# ゴルフ一打記録PWAアプリ 完全開発仕様書

## 1. プロジェクト概要
GitHub Pagesでホスティングし、iPhoneのSafariから「ホーム画面に追加」してネイティブアプリのように機能するPWA（Progressive Web App）を構築する。データ記録と外部出力（CSV）に特化し、アプリ内でのデータ分析機能は持たない。

## 2. 技術スタック
- フロントエンド: HTML5, Tailwind CSS (CDN経由), Vanilla JavaScript (フレームワーク不使用)
- ストレージ: `localStorage` (ブラウザローカル保存)
- PWA要件: `manifest.json`, Service Worker (`sw.js`)

## 3. ファイル構成
以下の4つのファイルを作成すること。
1. `index.html` (UIとJSロジックを統合したシングルページアプリケーション)
2. `manifest.json` (PWA設定ファイル)
3. `sw.js` (オフラインキャッシュ用Service Worker)
4. アイコン画像 (192x192, 512x512のダミーで可)

## 4. データ構造（localStorage）
最新の50ラウンド分を保存。50件を超えた場合は最も古いラウンドを削除するFIFO方式を採用。
現在進行中のラウンドを示す `active_round_id` を保持し、アプリ再起動時に入力状態を復元できるようにする。

```json
{
  "active_round_id": "2026-03-23T08:00:00Z",
  "rounds": [
    {
      "round_id": "2026-03-23T08:00:00Z",
      "date": "2026-03-23",
      "course_name": "座間ゴルフ練習場",
      "holes": [
        {
          "hole_num": 1,
          "par": 4,
          "shots": [
            {
              "shot_num": 1,
              "remaining_yards": 150,
              "club": "7I",
              "direction": "ストレート",
              "contact": "クリーン",
              "distance_feel": "適正",
              "memo": "やや前下がり。腕を真っ直ぐ出す意識。"
            }
          ]
        }
      ]
    }
  ]
}
```

## 5. 選択肢マスタデータ
- クラブ: 1W, 3W, 5W, 4U, 5I, 6I, 7I, 8I, 9I, PW, AW, 50°, 56°, SW, PT
- 方向・軌道: ストレート, 右プッシュ, スライス, 左ひっかけ, フック
- 打点: クリーン, トップ・チョロ, ダフリ, シャンク
- 距離感: 適正, ショート, オーバー

## 6. UI/UX設計（SPA構成）
JavaScriptによるDOM操作で、以下の3つの画面（View）を切り替える。iPhoneでのタップ操作に最適化すること（セレクトボックスは避け、大きめのボタングリッドを使用する）。

### 画面0：ホーム画面（初期画面）
- **新規ラウンド作成**:
  - 「新しいラウンドを開始」ボタンを配置。
  - タップでモーダルが開き、日付（初期値：今日）とコース名（テキスト入力）を入力させる。
  - 開始ボタン押下で新規データを作成し、「画面1」へ遷移。
- **ラウンド再開**:
  - `active_round_id` が存在する場合のみ、「現在進行中のラウンドを再開」ボタンを表示。タップで画面1へ遷移し、直前のホール・打数状態を復元する。
- **履歴・出力へ遷移**: 「過去の履歴とデータ出力」ボタン（画面2へ遷移）。

### 画面1：記録画面（入力特化）
- **ヘッダー**: 「ホームに戻る」ボタン（状態は保持される）と、現在のコース名を表示。
- **ホール管理**: ホール番号(1〜18)の増減ボタンと、Par(3, 4, 5)のボタントグル。切り替え時に該当ホールの現在の打数を自動計算して表示する。
- **ショット入力（1打ごとにリセット）**:
  - 現在の打数（自動算出）を表示。
  - **残りヤード**: `<input type="number">`で数字キーボードを起動（任意）。
  - **クラブ・方向・打点・距離感**: ボタングリッドで単一選択（必須項目）。
  - **メモ機能**: 「メモを追加」ボタンでテキストエリアを展開（任意）。
  - **記録ボタン**: 必須項目が全て選択されるまで非活性。押下時にデータを保存し、打数を+1して入力UIをリセット、トースト通知（「記録しました」）を出す。

### 画面2：履歴・出力画面
- **ヘッダー**: 「ホームに戻る」ボタン。
- **履歴参照**: プルダウンで過去のラウンド（日付＋コース名）を選択し、そのショット履歴を表形式で一覧表示（メモや残りヤードも表示）。
- **データ出力（CSV）機能**: 
  - 選択中のラウンドデータをCSV化してダウンロードさせる。
  - **【重要】CSVの仕様**:
    1. 文字化け防止のため、ファイルの先頭にBOM（`new Uint8Array([0xEF, 0xBB, 0xBF])`）を付与すること。
    2. ヘッダーは `Date,Course,Hole,Par,Shot,RemainingYards,Club,Direction,Contact,DistanceFeel,Memo` とすること。
    3. 各ショットの行データの先頭には、ラウンド階層が持つ `date` と `course_name` を必ず結合すること。
    4. `course_name` と `memo` はカンマや改行が含まれる可能性があるため、必ずダブルクォーテーションで囲み、内部のダブルクォーテーションはエスケープ処理（`""`）すること。

## 7. PWA対応ファイル要件
- `manifest.json`: `display: "standalone"`, `start_url: "./index.html"` を指定。
- `sw.js`: キャッシュ戦略（Cache First または Network First）を用いて `index.html` 等の静的アセットをオフライン利用可能にすること。
- `index.html`の `<head>` 内で manifest の読み込みと Service Worker の登録処理を記述すること。
